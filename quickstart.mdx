## Welcome to the Hotel Reservation System

This quickstart guide will help you set up and run the Hotel Reservation System from zero to a fully functional application. You'll have a working backend API and frontend interface managing hotel reservations in just a few minutes.

<Note>
  This system is built with Node.js, Express, and MySQL. Make sure you have these prerequisites installed before starting.
</Note>

## What You'll Build

By the end of this quickstart, you'll have:

- A REST API server running on port 4000
- MySQL database with tables for clients, rooms (habitaciones), and reservations
- A web interface to manage reservations, clients, and hotel rooms
- Full CRUD operations for all entities

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** (v14 or higher)
- **MySQL** (v5.7 or higher)
- **npm** (comes with Node.js)
- A code editor (VS Code recommended)
- A web browser

<Tip>
  Check your installations by running:
  ```bash
  node --version
  npm --version
  mysql --version
  ```
</Tip>

## Quick Setup Steps

<Steps>
  <Step title="Clone and Navigate">
    First, navigate to your backend directory:
    
    ```bash
    cd reservacionBackEnd/reservaionFronEnd
    ```
    
    This directory contains the Node.js/Express backend application.
  </Step>

  <Step title="Install Dependencies">
    Install all required npm packages:
    
    ```bash
    npm install
    ```
    
    This will install:
    - `express` - Web framework
    - `body-parser` - Request body parsing middleware
    - `cors` - Cross-Origin Resource Sharing
    - `mysql2` - MySQL database driver
    
    <Note>
      Installation typically takes 30-60 seconds depending on your internet connection.
    </Note>
  </Step>

  <Step title="Set Up the Database">
    Create the MySQL database and tables:
    
    ```sql
    CREATE DATABASE BDReservacion;
    USE BDReservacion;
    
    -- Create cliente table
    CREATE TABLE cliente (
        id INT AUTO_INCREMENT PRIMARY KEY,
        nombre VARCHAR(255) NOT NULL,
        correo VARCHAR(255) NOT NULL,
        telefono VARCHAR(20) NOT NULL
    );
    
    -- Create habitaciones table
    CREATE TABLE habitaciones (
        id INT AUTO_INCREMENT PRIMARY KEY,
        nombre VARCHAR(255) NOT NULL,
        tipo VARCHAR(100) NOT NULL,
        precio DECIMAL(10, 2) NOT NULL
    );
    
    -- Create reservaciones table
    CREATE TABLE reservaciones (
        id INT AUTO_INCREMENT PRIMARY KEY,
        folio VARCHAR(50) NOT NULL,
        nombrecliente VARCHAR(255) NOT NULL
    );
    ```
    
    <Tip>
      You can run these commands in MySQL Workbench or via the command line using `mysql -u root -p`.
    </Tip>
  </Step>

  <Step title="Configure Database Connection">
    Update the database credentials in `model/database.js`:
    
    ```javascript model/database.js
    const mysql = require('mysql2');
    
    const conexion = mysql.createConnection({
        host: 'localhost',
        user: 'root',
        password: '1234',  // Change this to your MySQL password
        database: 'BDReservacion'
    });
    
    conexion.connect((err) =>{
        if (err) throw err;
        console.log("Conexion exitosa a la base de datos")
    });
    
    module.exports = conexion;
    ```
    
    <Warning>
      Make sure to update the `password` field with your actual MySQL root password.
    </Warning>
  </Step>

  <Step title="Start the Server">
    Start the Express server:
    
    ```bash
    node server.js
    ```
    
    You should see:
    ```
    Servidor iniciado en el puerto 4000
    Conexion exitosa a la base de datos
    ```
    
    <Note>
      The server runs on port 4000 by default. Make sure this port is not in use by another application.
    </Note>
  </Step>

  <Step title="Open the Frontend">
    Open the frontend files in your browser. Navigate to the parent directory:
    
    ```bash
    cd ..
    ```
    
    Then open any of these HTML files:
    - `index.html` - Client management interface
    - `habitaciones.html` - Room management interface
    - `reservaciones.html` - Reservation management interface
    - `Menu.html` - Navigation menu
    
    <Tip>
      You can open these files directly in your browser or use a local development server like Live Server in VS Code.
    </Tip>
  </Step>
</Steps>

## Available API Endpoints

Your server exposes the following REST API endpoints:

### Clients Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/clientes` | Get all clients |
| POST | `/clientes` | Create a new client |
| PUT | `/clientes/:id` | Update a client |
| DELETE | `/clientes/:id` | Delete a client |

### Rooms (Habitaciones) Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/habitaciones` | Get all rooms |
| POST | `/habitaciones` | Create a new room |
| PUT | `/habitaciones/:id` | Update a room |
| DELETE | `/habitaciones/:id` | Delete a room |

### Reservations Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/reservaciones` | Get all reservations |
| POST | `/reservaciones` | Create a new reservation |
| PUT | `/reservaciones/:id` | Update a reservation |
| DELETE | `/reservaciones/:id` | Delete a reservation |

## Common Issues and Solutions

<AccordionGroup>
  <Accordion title="Server won't start - Port already in use">
    If you see an error about port 4000 being in use:
    
    1. Find the process using the port:
       ```bash
       # On Linux/Mac
       lsof -i :4000
       
       # On Windows
       netstat -ano | findstr :4000
       ```
    
    2. Kill the process or change the port in `server.js`:
       ```javascript
       const PORT = 5000; // Change to any available port
       ```
  </Accordion>

  <Accordion title="Database connection failed">
    If you see "Error connecting to database":
    
    1. Verify MySQL is running:
       ```bash
       # On Linux
       sudo systemctl status mysql
       
       # On Windows
       services.msc (look for MySQL)
       ```
    
    2. Check your credentials in `model/database.js`
    3. Ensure the database `BDReservacion` exists
  </Accordion>

  <Accordion title="CORS errors in browser console">
    The server already has CORS enabled in `server.js`:
    
    ```javascript
    const cors = require("cors");
    app.use(cors());
    ```
    
    If you still see CORS errors, ensure the frontend is making requests to `http://localhost:4000`.
  </Accordion>

  <Accordion title="npm install fails">
    Try clearing the npm cache and reinstalling:
    
    ```bash
    npm cache clean --force
    rm -rf node_modules
    rm package-lock.json
    npm install
    ```
  </Accordion>
</AccordionGroup>

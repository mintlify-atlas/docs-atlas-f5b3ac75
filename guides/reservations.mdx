## Overview

The Reservation Management module is the core functionality of the Hotel Reservation System. It allows staff to create, update, and track guest reservations with a simple folio-based system.

<Note>
  Reservations consist of a unique folio number and the client's name. The API endpoint is `http://localhost:4000/reservaciones`.
</Note>

## Reservation Data Structure

Each reservation record contains:

| Field | Type | Description | Example |
|-------|------|-------------|----------|
| `id` | Integer | Auto-generated unique identifier | 1 |
| `folio` | String | Unique reservation folio number | "F-2024-001" |
| `nombrecliente` | String | Client's full name | "Juan Pérez" |

<Warning>
  The current implementation stores basic reservation data. In production, you might want to add fields like check-in date, check-out date, room ID, and total amount.
</Warning>

#### Loading All Reservations

```javascript
async function loadReservaciones() {
    try {
        const response = await fetch(URL_RESERVACIONES);

            html += `

                <td>${reservacion.folio}</td>

            </tr>`;

        console.error('Error al cargar las reservaciones:', error);
    }
}
```

- Generates table rows with folio and client name
- Includes Bootstrap modal triggers for updates
- Error logging for debugging

### Updating Reservations

Reservation updates use a two-step process: selection and submission.

#### Step 1: Select Reservation

```javascript
function seleccionarReservacion(id) {
    reservacionSeleccionada = reservaciones.find(res => res.id === id);
    document.getElementById("folioReservacionU").value = reservacionSeleccionada.folio;
    document.getElementById("clienteReservacionU").value = reservacionSeleccionada.nombrecliente;
}
```

**Key Points:**
- Uses `find()` to locate reservation by ID
- Stores selected reservation in global variable
- Pre-fills update form with current values
- Triggered when user clicks "Actualizar" button

<Note>
  The Bootstrap modal trigger is built into the button's `data-bs-toggle` and `data-bs-target` attributes, so the modal opens automatically after selection.
</Note>

#### Step 2: Submit Update

```javascript
async function actualizarReservacion() {
    const folio = document.getElementById("folioReservacionU").value;
    const nombrecliente = document.getElementById("clienteReservacionU").value;

    const data = { folio, nombrecliente };

    try {
        const response = await fetch(`${URL_RESERVACIONES}/${reservacionSeleccionada.id}`, {
            method: "PUT",
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify(data)
        });

        if (!response.ok) throw new Error('Error al actualizar reservación');

        loadReservaciones();
        limpiarFormulario('updateReservacion');

    } catch (error) {
        console.error('Error al actualizar reservación:', error);
    }
}
```

**Update Process:**
1. Extract updated values from form
2. Get reservation ID from `reservacionSeleccionada`
3. Send PUT request to `/reservaciones/:id`
4. Reload reservations list
5. Clear form and close modal

#### Backend Update Handler

```javascript
exports.updateDataReservaciones = (req, res) => {
    const id = req.params.id;
    const { folio, nombrecliente } = req.body;
    const SQL = "UPDATE `reservaciones` SET folio=?, nombrecliente=? WHERE id=?";
    conexion.query(SQL, [folio, nombrecliente, id], (err, results) => {
        if (err) throw err;
        if (results.affectedRows === 0) {
            res.status(404).json({ message: "Reservación no encontrada" });
        } else {
            res.json({ message: "Reservación actualizada correctamente" });
        }
    });
};
```

**Route:** `PUT /reservaciones/:id`

**Features:**
- Checks if any rows were affected
- Returns 404 if reservation doesn't exist
- Returns success message on update

### Deleting Reservations

Delete reservations permanently from the system.

```javascript
async function eliminarReservacion(id) {
    try {
        const response = await fetch(`${URL_RESERVACIONES}/${id}`, { method: "DELETE" });
        if (!response.ok) throw new Error('Error al eliminar reservación');

        loadReservaciones();

    } catch (error) {
        console.error('Error al eliminar reservación:', error);
    }
}
```

<Warning>
  Deletion is permanent. Consider implementing soft deletes or requiring confirmation dialogs for better data safety.
</Warning>

#### Backend Delete Handler

```javascript
exports.deleteDataReservaciones = (req, res) => {
```

## Utility Functions

### Clearing Forms

The `limpiarFormulario()` function clears input fields and closes modals:

```javascript
function limpiarFormulario(modalId) {
    document.getElementById(modalId).querySelectorAll("input").forEach(input => input.value = '');
    const modal = bootstrap.Modal.getInstance(document.getElementById(modalId));
    modal.hide();
}
```

**Usage:**
- `limpiarFormulario('formDataReservacion')` - Clear new reservation form
- `limpiarFormulario('updateReservacion')` - Clear update form

**Process:**
1. Select all input elements within the modal
2. Clear each input's value
3. Get the Bootstrap Modal instance
4. Hide the modal programmatically

## Best Practices

<Accordion title="Folio Generation">
  Implement automatic folio generation to ensure uniqueness:
  
  ```javascript
  function generateFolio() {
      const date = new Date();
      const year = date.getFullYear();
      const month = String(date.getMonth() + 1).padStart(2, '0');
      const day = String(date.getDate()).padStart(2, '0');
      const random = Math.floor(Math.random() * 10000).toString().padStart(4, '0');
      
      return `F-${year}${month}${day}-${random}`;
  }
  
  // Usage in form
  async function guardarReservacionWithAutoFolio() {
      const folio = generateFolio();
      const nombrecliente = document.getElementById("clienteReservacion").value;

      const data = { folio, nombrecliente };

      try {
          const response = await fetch(URL_RESERVACIONES, {
              method: "POST",
              headers: { 'Content-Type': 'application/json' },
              body: JSON.stringify(data)
          });

          if (!response.ok) throw new Error('Error al guardar reservación');

          loadReservaciones();
          limpiarFormulario('formDataReservacion');

      } catch (error) {
          console.error('Error al guardar reservación:', error);
      }
  }
  ```
  
  **Folio Format:** `F-YYYYMMDD-XXXX`
  - F: Prefix for "Folio"
  - YYYYMMDD: Current date
  - XXXX: Random 4-digit number
</Accordion>

```javascript
function searchReservaciones(query) {
    const lowerQuery = query.toLowerCase();
    
    const results = reservaciones.filter(reservacion =>
```

```javascript
function sortReservaciones(field, order = 'asc') {
    const sortedData = [...reservaciones].sort((a, b) => {
        if (order === 'asc') {
            return a[field].localeCompare(b[field]);
        } else {
            return b[field].localeCompare(a[field]);
        }

    renderReservacionTable(sortedData);
```

### Export Reservations

Export reservations to CSV format:

```javascript
function exportToCSV() {
    let csv = 'Folio,Cliente\n';
    
    reservaciones.forEach(reservacion => {
        csv += `"${reservacion.folio}","${reservacion.nombrecliente}"\n`;
    });
    
    const blob = new Blob([csv], { type: 'text/csv' });
    const url = window.URL.createObjectURL(blob);

    a.href = url;
    a.download = `reservaciones_${new Date().toISOString().split('T')[0]}.csv`;
    a.click();
    window.URL.revokeObjectURL(url);
}
```

## Database Schema

The `reservaciones` table structure:

```sql
CREATE TABLE `reservaciones` (
  `id` INT AUTO_INCREMENT PRIMARY KEY,
  `folio` VARCHAR(100) NOT NULL UNIQUE,
  `nombrecliente` VARCHAR(255) NOT NULL
);
```

<Note>
  Consider adding an index on the `folio` column for faster lookups and ensuring uniqueness with a UNIQUE constraint.
</Note>

## Enhanced Schema (Recommended)

For production systems, consider this enhanced schema:

```sql
CREATE TABLE `reservaciones` (
  `id` INT AUTO_INCREMENT PRIMARY KEY,
  `folio` VARCHAR(100) NOT NULL UNIQUE,
  `cliente_id` INT NOT NULL,
  `habitacion_id` INT NOT NULL,
  `fecha_entrada` DATE NOT NULL,
  `fecha_salida` DATE NOT NULL,
  `total` DECIMAL(10, 2) NOT NULL,
  `estado` ENUM('pendiente', 'confirmada', 'cancelada', 'completada') DEFAULT 'pendiente',
  `fecha_creacion` TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (`cliente_id`) REFERENCES `cliente`(`id`),
  FOREIGN KEY (`habitacion_id`) REFERENCES `habitaciones`(`id`)
);
```

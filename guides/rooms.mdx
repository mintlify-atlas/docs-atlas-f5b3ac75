## Overview

The Room Management module enables hotel administrators to manage room inventory, including room numbers, types, and pricing. This guide covers all room operations with practical examples from the application.

<Note>
  Room data includes room number (nombre), type (tipo), and price (precio). All operations use the REST API endpoint `http://localhost:4000/habitaciones`.
</Note>

## Room Data Structure

Each room record contains the following fields:

| Field | Type | Description | Example |
|-------|------|-------------|----------|
| `id` | Integer | Auto-generated unique identifier | 1 |
| `nombre` | String | Room number or identifier | "101", "Suite Presidential" |
| `tipo` | String | Room type or category | "Single", "Double", "Suite" |
| `precio` | Decimal/String | Price per night | "150.00" |

## CRUD Operations

### Reading Room Data

The application loads all rooms automatically when the page initializes.

#### Loading All Rooms

```javascript
async function loadDataHabitacion() {
    try {
        const response = await fetch(URL);
        data = await response.json();

        let html = '';
        data.forEach(habitacion => {
            html += `
            <tr>
                <td>${habitacion.id}</td>
                <td>${habitacion.nombre}</td>
                <td>${habitacion.tipo}</td>
                <td>${habitacion.precio}</td>
                <td>
                    <button type="button" class="btn btn-primary btn-sm" onclick="modificarDataHabitacion(${habitacion.id})">Actualizar</button>
                    <button type="button" class="btn btn-danger btn-sm" onclick="deleteDataHabitacion(${habitacion.id})">Eliminar</button>
                </td>
            </tr>`;
        });

        T_DATA_HABITACION.innerHTML = html;

    } catch (error) {
        console.error(error);
    }
}
```

**Key Features:**
- Asynchronous data fetching with `fetch()` API
- Dynamic HTML generation for table rows
- Error handling with try-catch
- Inline action buttons with room ID binding

### Updating Existing Rooms

Room updates follow a two-phase pattern: selection and submission.

#### Phase 1: Opening the Update Dialog

```javascript
function modificarDataHabitacion(id) {
    const dataHabitacion = data.find(habitacion => habitacion.id === id);
    document.getElementById("roomNameU").value = dataHabitacion.nombre;
    document.getElementById("roomTypeU").value = dataHabitacion.tipo;
    document.getElementById("roomPriceU").value = dataHabitacion.precio;

    const dialog = document.getElementById("formDataHabitacion");
    dialog.dataset.habitacionId = id;
    dialog.showModal();
}
```

**Key Features:**
- Uses JavaScript `find()` method to locate room by ID
- Pre-populates form fields with current values
- Stores room ID in the dialog's dataset for later retrieval
- Uses native HTML5 `<dialog>` element with `showModal()`

#### Phase 2: Submitting Updates

```javascript
async function actualizarDataHabitacion() {
    const dialog = document.getElementById("formDataHabitacion");
    const id = dialog.dataset.habitacionId;

    const dataHabitacion = {
        nombre: document.getElementById("roomNameU").value,
        tipo: document.getElementById("roomTypeU").value,
        precio: document.getElementById("roomPriceU").value
    };

    try {
        const response = await fetch(`${URL}/${id}`, {
            method: "PUT",
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify(dataHabitacion)
        });

        if (!response.ok) throw new Error('Error al actualizar habitación');

        dialog.close();
        loadDataHabitacion();

    } catch (error) {
        console.error(error);
    }
}
```

**Update Flow:**
1. Retrieve room ID from dialog dataset
2. Collect updated values from form
3. Send PUT request to `/habitaciones/:id`
4. Close dialog on success
5. Refresh room list to show changes

#### Backend Update Handler

```javascript
exports.updateDataHabitaciones = (req, res) => {
    const id = req.params.id;
    const { nombre, tipo, precio  } = req.body;
    const SQL = "UPDATE `habitaciones` SET nombre=?, tipo=?, precio=? WHERE id=?";
    conexion.query(SQL, [nombre, tipo, precio, id], (err, results) => {
        if (err) throw err;
        res.json({ message: "habitacion actualizado" });
    });
};
```

**Route:** `PUT /habitaciones/:id`

<Warning>
  Ensure price values are properly validated. Consider using decimal type validation to prevent invalid price entries.
</Warning>

### Deleting Rooms

Room deletion permanently removes the record from the database.

```javascript
async function deleteDataHabitacion(id) {
    try {
        const response = await fetch(`${URL}/${id}`, { method: "DELETE" });
        if (!response.ok) throw new Error('Error al eliminar habitación');

        loadDataHabitacion();

    } catch (error) {
        console.error(error);
    }
}
```

<Warning>
  Deleting a room that has active reservations may cause data integrity issues. Implement validation to check for existing reservations before deletion.
</Warning>

#### Backend Delete Handler

```javascript
exports.deleteDataHabitaciones = (req, res) => {
    const id = req.params.id;
    const SQL = "DELETE FROM `habitaciones` WHERE id=?";
    conexion.query(SQL, id, (err, results) => {
        if (err) throw err;
        res.json({ message: "habitacion eliminado" });
    });
};
```

**Route:** `DELETE /habitaciones/:id`

## Room Types and Pricing

### Common Room Types

The system supports flexible room type definitions. Here are common examples:

| Room Type | Description | Typical Price Range |
|-----------|-------------|---------------------|
| Single | One bed, basic amenities | $80 - $150 |
| Double | Two beds or one double bed | $120 - $200 |
| Suite | Multiple rooms, premium amenities | $250 - $500 |
| Presidential Suite | Luxury suite with full services | $500+ |
| Economy | Basic room, budget-friendly | $50 - $100 |

<Note>
  Room types are stored as free-text fields, allowing flexibility in naming conventions.
</Note>

<Accordion title="Room Number Validation">


### Sorting Rooms

```javascript
function sortRooms(field, order = 'asc') {
    const sortedData = [...data].sort((a, b) => {
        if (field === 'precio') {
            // Numeric sort for prices
            return order === 'asc' 
                ? parseFloat(a.precio) - parseFloat(b.precio)
                : parseFloat(b.precio) - parseFloat(a.precio);
        } else {
            // String sort for other fields
            return order === 'asc'
                ? a[field].localeCompare(b[field])
                : b[field].localeCompare(a[field]);
        }
    });
    
    // Re-render table with sorted data
    renderRoomTable(sortedData);
}
```

### Search Functionality

```javascript
function searchRooms(query) {
    const lowerQuery = query.toLowerCase();
    
    const results = data.filter(habitacion =>
```

## Integration with Reservations

Rooms are linked to reservations. Before implementing room management, consider:

<Warning>
  - Do not delete rooms with active reservations
  - Price changes affect future reservations only
  - Room number changes may break existing references
</Warning>

## Introduction

The Hotel Reservation System provides a RESTful API built with Express.js for managing hotel operations. The API supports CRUD operations for clients, rooms, and reservations.

<Note>
  The API runs on `http://localhost:4000` by default and uses JSON for all request and response payloads.
</Note>

## Base URL

All API endpoints are relative to the base URL:

```
http://localhost:4000
```

For production deployments, replace `localhost:4000` with your server's domain and port.

### Server Configuration

The API server is configured in `server.js:1-28`:

```javascript server.js
const express = require('express');
const bodyParser = require('body-parser');
const clientesRouter = require('./router/clientes.Router');
const habitacionRouter = require('./router/habitaciones.Router');
const reservacionRouter = require('./router/reservaciones.Router');
const cors = require("cors");

const app = express();
const PORT = 4000;

// Middleware
app.use(cors());
app.use(bodyParser.json());
app.use(clientesRouter);
app.use(habitacionRouter);
app.use(reservacionRouter);

// Start server
app.listen(PORT, () => {
    console.log(`Servidor iniciado en el puerto ${PORT}`);
});
```

## Request Format

### Headers

All POST and PUT requests must include the `Content-Type` header:

```http
Content-Type: application/json
```

### Request Body

Request bodies must be valid JSON. Example:

```json
{
  "nombre": "John Doe",
  "correo": "john@example.com",
  "telefono": "555-0123"
}
```

## Response Format

### Success Responses

Successful requests return JSON data with appropriate HTTP status codes:

**List/Get Requests (200 OK)**
```json
[
  {
    "id": 1,
    "nombre": "John Doe",
    "correo": "john@example.com",
    "telefono": "555-0123"
  }
]
```

**Create/Update/Delete Requests (200 OK)**
```json
{
  "message": "Cliente agregado"
}
```

### Error Responses

Errors are thrown and handled by Express's error handling middleware. Common error scenarios:

<Accordion title="Database Connection Errors">
  If the MySQL database is unavailable, requests will fail with a database connection error.
  
  **Solution**: Ensure MySQL is running and connection settings in `model/database.js` are correct.
</Accordion>

<Accordion title="Invalid JSON Payload">
  Malformed JSON in request body will result in parsing errors.
  
  **Solution**: Validate JSON syntax before sending requests.
</Accordion>

<Accordion title="Missing Required Fields">
  SQL constraints will cause errors if required fields are missing.
  
  **Solution**: Include all required fields as documented in each endpoint.
</Accordion>

## HTTP Methods

The API uses standard HTTP methods:

| Method | Purpose | Example |
|--------|---------|----------|
| `GET` | Retrieve resources | Get all clients |
| `POST` | Create new resource | Add new client |
| `PUT` | Update existing resource | Update client details |
| `DELETE` | Remove resource | Delete a client |

## CORS Configuration

Cross-Origin Resource Sharing (CORS) is enabled for all origins:

```javascript server.js lines
const cors = require("cors");
app.use(cors());
```

<Warning>
  In production, configure CORS to allow only trusted origins for security.
</Warning>

### Production CORS Example

```javascript
const cors = require('cors');

const corsOptions = {
  origin: 'https://yourdomain.com',
  methods: ['GET', 'POST', 'PUT', 'DELETE'],
  credentials: true
};

app.use(cors(corsOptions));
```

## Database Connection

The API connects to MySQL using the `mysql2` package. Configuration in `model/database.js:1-22`:

```javascript database.js
const mysql = require('mysql2');

const conexion = mysql.createConnection({
    host: 'localhost',
    user: 'root',
    password: '1234',
    database: 'BDReservacion'
});

conexion.connect((err) =>{
    if (err) throw err;
    console.log("Conexion exitosa a la base de datos")
});

module.exports = conexion;
```

## Quick Start Example

Here's a complete example of interacting with the API:

<CodeGroup>

```bash curl
# Get all clients
curl http://localhost:4000/clientes

# Create a new client
curl -X POST http://localhost:4000/clientes \
  -H "Content-Type: application/json" \
  -d '{"nombre":"Jane Smith","correo":"jane@example.com","telefono":"555-0199"}'

# Update a client
curl -X PUT http://localhost:4000/clientes/1 \
  -H "Content-Type: application/json" \
  -d '{"nombre":"Jane Doe","correo":"jane@example.com","telefono":"555-0199"}'

# Delete a client
curl -X DELETE http://localhost:4000/clientes/1
```

```javascript fetch
// Get all clients
fetch('http://localhost:4000/clientes')
  .then(response => response.json())
  .then(data => console.log(data));

// Create a new client
fetch('http://localhost:4000/clientes', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    nombre: 'Jane Smith',
    correo: 'jane@example.com',
    telefono: '555-0199'
  })
})
  .then(response => response.json())
  .then(data => console.log(data));

// Update a client
fetch('http://localhost:4000/clientes/1', {
  method: 'PUT',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    nombre: 'Jane Doe',
    correo: 'jane@example.com',
    telefono: '555-0199'
  })
})
  .then(response => response.json())
  .then(data => console.log(data));

// Delete a client
fetch('http://localhost:4000/clientes/1', {
  method: 'DELETE'
})
  .then(response => response.json())
  .then(data => console.log(data));
```

</CodeGroup>

## Rate Limiting

<Info>
  Currently, the API does not implement rate limiting. For production use, consider adding rate limiting middleware.
</Info>

### Adding Rate Limiting (Optional)

```javascript
const rateLimit = require('express-rate-limit');

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 100 // limit each IP to 100 requests per windowMs
});

app.use(limiter);
```
